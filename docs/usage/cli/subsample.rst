===============
augur subsample
===============

   :local:

Terminology
===========

- sample: A set of sequence ids selected from the input according to filtering
  and subsampling rules (e.g., minimum/maximum date, minimum/maximum sequence
  length, sample size). A sample is defined by these options.

Implementation details
======================

- Each sample is generated by an independent ``augur filter`` invocation. Samples
  run in parallel when ``--nthreads`` > 1.
- Each invocation writes a text file containing the selected sequence ids for
  that sample.
- The output dataset is produced by a final ``augur filter`` call that subsets the
  inputs using the union of all sample id files.
- CLI and YAML config options map closely to augur filter options, see below.

Note that the following ``augur filter`` options are not supported:

- ``--priority``
- ``--output-log``
- ``--output-group-by-sizes``
- ``--output-strains``
- ``--empty-output-reporting``
